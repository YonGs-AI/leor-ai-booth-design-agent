# 公众号稿件

## 标题

展台设计，现在可以让 Agent 直接调用 LEOR 了

## 摘要

LEOR AI展台设计 Agent 让 Codex、Claude Code、OpenClaw、WorkBuddy、QClaw 等 Agent 工具通过对话调用 LEOR。从需求整理、账号授权到生成和保存，用户不用在多个软件之间来回倒图。

## 正文

做展台方案时，真正费时间的往往不是打开某一个生图页面。

业务员先拿到一段零散需求，要问品牌、展品、尺寸、墙体和开口；设计师还要找 Logo、找参考图、理风格，出图后再把结果放回项目里。环节不难，但细碎。

LEOR 现在把这条工作链接给了 Agent。

### 它不是多一个聊天页面

LEOR AI展台设计 Agent 支持 Codex、Claude Code、OpenClaw、WorkBuddy、QClaw 等 Agent 工具，让 AI 通过对话调用 LEOR 完成展台设计、生图、改图和项目资产保存。

这里的关键不是“和 AI 聊几句”，而是 Agent 能真正调用 LEOR 的工具。

你可以在自己熟悉的 Agent 里说：

> 我要做一个展台设计，先把需要的资料一次列给我，不要直接生图。

Agent 应该先收集品牌名称、展示产品、宽度、深度、高度、前后左右的墙体开口、风格、色彩、造型、材料和功能区。Logo 和参考图不是硬性必填，但它必须问你有没有。有 Logo 就尽量传，有参考图也一起给，结果会更靠近实际项目。

### 扣积分前，先把话说清楚

展台设计不是报一个颜色就能开始。

如果你说“帮我找几个橙色展台参考”，Agent 应该继续找参考，而不是擅自扣积分生成一张橙色展台。

如果你真的要出图，它会在生成前把已收集的需求整理成一份清单，等你确认后再调用 LEOR。默认每次只生成 1 张，避免因为 Agent 理解偏差而一次消耗多份积分。

### 图片可以直接发给 Agent

需要带图的任务，普通用户不应该被要求自己找图床、拿图片链接或转 Base64。

比如你发一张水果、产品、包装、Logo 或材质图，说“用这张图做万物成展”。Agent 把这张图交给 LEOR，LEOR 会把它存到当前授权用户的 Agent 项目里，然后直接执行这次创意推导。

同样，你也可以发一张现有展台图，让它改图、生成展位分镜、材质看板、海报或旋转视角。

### 你在 Agent 里做，结果仍然回到 LEOR

第一次使用时，Agent 会返回 LEOR 账号授权页。你登录并确认后，后续扣分、任务和项目资产都归属这个 LEOR 账号。

这点很重要。你今天在 Agent 对话里出的图，过几天回到 LEOR，仍然可以在自己的项目里继续查看和处理。

LEOR AI展台设计 Agent 已正式开放使用。它的价值不是取代展台设计师，而是让展台设计搭建团队少做一些软件间的搬运，把更多时间放在需求判断和方案推进上。
