# 知乎问答稿

## 问题

有没有可以让 AI Agent 直接做展台设计的工具？

## 回答

有，但先要分清两件事：一个普通 AI 聊天页面会写展台提示词，不等于它真的能够调用展台设计工具。

要让 Agent 真正完成展台设计、生图、改图和保存项目资产，需要它能连接一个外部工具服务。MCP 就是目前常见的连接方式之一。

LEOR 做的是这后半段。

LEOR AI展台设计 Agent 支持 Codex、Claude Code、OpenClaw、WorkBuddy、QClaw 等 Agent 工具，让 AI 通过对话调用 LEOR 完成展台设计、生图、改图和项目资产保存。其他支持 MCP 或外部工具调用的 Agent 环境，也可以按说明接入。

### 它解决的不只是“生一张图”

展台设计的输入不是一句“做得高级一点”。

一个能用的展台设计需求，至少要说清品牌/企业名称、展示产品、宽度、深度、高度、前后左右四面哪里是墙、哪里是开口，以及风格、颜色、造型、材料和功能区。输出时还要选模型、分辨率和画面比例。

Logo 和参考图是可选资产，但这不等于 Agent 可以不问。品牌需求明确时，Logo 应该尽量提供；有参考图时，生成结果也更容易靠近目标。

因此，一个设计正常的 Agent 工作流程应该是：

1. 一次性列出必填和可选资料。
2. 补齐需求，不因为收到几个回答就提前生图。
3. 把完整需求汇总给用户。
4. 用户明确确认后，再调用 LEOR 扣积分生成。
5. 默认只生成 1 张，需要多张时按单张任务依次执行。

这个步骤不只是为了设计质量，也是为了保护用户积分。如果用户说“帮我找几个橙色展台参考”，Agent 应该继续搜索参考，不应该把“橙色”当成新的生成指令。

### 哪些功能必须带图？

新建展台设计可以从完整的文字需求开始，同时建议提供 Logo 和参考图。

其他多数图像功能则必须有源图片：

- 修改展台图：需要现有展台图和明确改图要求。
- 万物成展：需要一张任意视觉来源，通常不是现有展台图。它可以是动物、水果、产品、包装、Logo、色卡、材料或纹理，用于展台设计前期创意推导。
- 展位分镜、材质看板、潮流海报、简约海报和玻璃质感海报：都需要一张现有展台图。
- 旋转视角：需要一张现有展台图，在 LEOR 已设定的 14 个额外视角中选择，不是自由摄像机控制。

对普通用户来说，图片应该可以直接发到 Agent 对话里。用户不需要自己拿图片 URL，也不需要转 Base64。Agent 负责把这张图交给 LEOR，LEOR 再完成存入项目、生成和结果返回。

### 用 Agent 出的图，为什么还要回到 LEOR 项目？

因为 Agent 对话是调用入口，不应该变成另一个孤立图库。

用户第一次连接 LEOR 时，会打开 LEOR 授权页，登录并确认授权。后续生成的图、任务记录和项目资产都归属这个账号。这样无论用户是从 Agent 开始，还是从 LEOR 网页开始，最后都在同一个项目里继续处理。

### 它与普通 AI 生图有什么不同？

普通 AI 生图工具也能生成一张看起来像展台的图。但展台工作不只需要“像”，还要考虑品牌、展品、尺寸、开口、墙体、造型、材料和后续提案用途。

LEOR AI展台设计 Agent 的重点，是把这些展台业务规则交给 Agent，并用生成前的资料门禁防止误操作。它不替代设计师，也不承诺直接生成施工图。它更适合需求整理、展台设计图生成、方案对比、改图和提案资产准备。

### 怎么安装？

在支持 MCP 的 Agent 工具里添加 LEOR 的远程 MCP 地址，然后对 Agent 说“连接我的 LEOR 账号”。Agent 会返回授权链接，用户登录 LEOR 并确认后即可使用。

普通用户不需要复制 token，不需要查看 `.env`，也不需要修改网关代码。

### 模型和积分怎么算？

Agent 对话中只使用 LEOR 的三个公开中文名称：快速渲染模型、专业渲染模型和全能渲染模型。三个模型都支持 1K、2K 和 4K。

快速渲染模型的 1K/2K/4K 分别是 3/4/6 积分；专业渲染模型是 4/5/7 积分；全能渲染模型是 6/8/10 积分。默认选择是专业渲染模型 + 2K，生成 1 张新展台设计图通常消耗 5 积分。

万物成展、展位分镜、材质看板和三类海报工具，在所选模型和分辨率的基础积分上加 3 积分。旋转视角也是扣积分功能，Agent 在调用前应该明确告知。

这些费用都扣在当前授权的 LEOR 账号上。用户也可以直接问 Agent：“查一下我 LEOR 账号还剩多少积分。”这是只读查询，不会扣分。
