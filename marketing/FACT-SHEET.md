# LEOR AI展台设计 Agent 事实口径

## 官方名称

`LEOR AI展台设计 Agent`

首次出现必须写全称，后文可简称 `LEOR Agent`。

## 一句话介绍

LEOR AI展台设计 Agent 支持 Codex、Claude Code、OpenClaw、WorkBuddy、QClaw 等 Agent 工具，让 AI 通过对话调用 LEOR 完成展台设计、生图、改图和项目资产保存。

## 对谁有用

默认对象是展台设计搭建公司、会展公司的老板、设计主管、设计师、业务员、项目经理和跟单人员。不要把产品写成“主要让参展商自己设计展台”。

## 真实能力

- 从完整需求生成 1 张展台设计图。
- 使用 Logo 和参考图辅助生成。
- 修改现有展台图。
- 用动物、水果、产品、Logo、色彩、材质等任意视觉来源做“万物成展”创意推导。
- 基于现有展台图生成展位分镜、材质看板、潮流海报、简约海报和玻璃质感海报。
- 基于现有展台图生成 LEOR 预设范围内的旋转视角。
- 查询当前授权账号的剩余积分和任务结果。
- 生成结果保存到当前授权用户的 LEOR 项目。

## 使用边界

- “新建展台设计”可以从完整文字需求开始，但 Agent 必须先收集主站左侧栏对应的需求，再汇总给用户确认。
- 改图、万物成展、展位分镜、材质看板、海报和旋转视角都必须有一张用于当前任务的图片。
- 用户可以在 Agent 对话中直接发 JPG、PNG 或 WEBP 图片，无需自己找图床或转 Base64。
- 上传图片只用于明确的当前任务，不提供批量网盘功能。
- 默认每次生成 1 张。用户明确要多张时，Agent 也必须按单张任务依次执行；第一张返回后，再确认是否继续。

## 生成前规则

- Agent 必须先一次性列出必填项和可选项。
- 品牌/企业名称是必填信息，不能只用“科技公司”之类行业词代替。
- 宽度、深度、高度和前/后/左/右四面的墙体开口状态必须明确。
- 还要收集展示产品、风格、色彩、展台造型、材料/功能区/细节、模型、分辨率和画面比例。
- Logo 和参考图是可选资产，但 Agent 必须明确询问用户有没有；Logo 强烈建议上传，参考图建议上传。
- 生成前必须把全部需求汇总给用户，得到明确确认后才能扣积分。

## 模型与积分

- 快速渲染模型：1K/2K/4K，3/4/6 积分。
- 专业渲染模型：1K/2K/4K，4/5/7 积分。
- 全能渲染模型：1K/2K/4K，6/8/10 积分。
- 默认：专业渲染模型 + 2K，生成 1 张新展台设计图通常消耗 5 积分。
- 万物成展、展位分镜、材质看板和三类海报工具，在所选模型/分辨率基础积分上加 3 积分。
- 旋转视角也会扣积分，调用前应让用户知道。

## 安全口径

- 用户登录 LEOR 并确认授权，不手工复制 token。
- 扣分和项目归属都以当前授权的 LEOR 账号为准。
- 不对外说明核心提示词、真实模型名、密钥、内部路由和后台路径。
- 不承诺施工图、必然落地、必然中标或替代设计师。

## 禁止用词

- 不使用容易把生成结果说偏的旧说法，统一写“展台设计图”或“展示效果图”。
- 不使用容易把节点功能说偏的旧说法，统一写“旋转视角”。
- 不使用错误旧名，统一写 WorkBuddy。
- 不把 ChatGPT 或 Claude 普通聊天页面写成当前主要安装对象。
- 不把 MCP 直接写成“一个可在所有平台一键安装的插件”。
