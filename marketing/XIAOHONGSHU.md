## 正文

做展台方案的人应该都熟悉这个过程：客户把需求发过来，我们先把信息整理一遍，再去找 Logo、找参考图、打开设计工具，出完图还要放回项目里。

这组图想讲的，就是这些步骤怎么在 Agent 对话里串起来。👀

现在可以让 Codex、Claude Code、OpenClaw、WorkBuddy、QClaw 这类 Agent 连接 LEOR。你说“我要做一个展台设计”，它不应该马上生图，而是先把需要的信息一次列给你。

必须问清楚的不只是尺寸和颜色，还包括品牌名称、展示产品、宽度/深度/高度、前后左右的墙体开口、风格、造型、材料和功能区。Logo 强烈建议上传，有参考图也尽量一起发。🧩

资料齐了以后，Agent 要先汇总给你确认，然后才能调用 LEOR 扣积分生图。默认只生成 1 张，避免它理解错了还连续扣分。⚠️

带图任务也一样。你可以直接发一张展台图，让它改图、生成材质看板、展位分镜、海报或旋转视角；也可以发一张水果、动物、产品、Logo 或材质图，做“万物成展”的创意推导。

不用自己找图床，也不用管图片链接和 Base64。你只需要告诉 Agent 这张图要用来做什么。

生成的图会保存到当前授权的 LEOR 项目里。今天在 Agent 里出图，过几天回到 LEOR，项目资产还在。📌

它不是施工图工具，也不是替代设计师。它解决的是展台方案前期那些信息整理、出图、改图和资产保存的来回切换。

做展台的薯友，你最想先让 Agent 帮你跑哪一步？

## 轮播顺序

1. 封面：`AI展台设计，现在 Agent 可以直接做了`，主画面必须是完整展台设计图。
2. 对话截图：用户说“我要做一个展台设计”，Agent 返回完整资料清单。
3. 需求汇总：展示品牌、尺寸、开口、风格、造型、材料、Logo 和参考图决定。
4. 生成前确认：用户确认后再扣积分，默认 1 张。
5. 主结果：完整展台设计图，页内只保留必要说明。
6. 带图工作流：用户发图 + 说明任务 -> LEOR 自动存入项目并执行。
7. 延展结果：展位分镜/材质看板/海报/旋转视角拼图。
8. 项目归属：Agent 里生成，LEOR 项目里继续保存。
